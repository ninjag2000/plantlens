# Инструкция по перезапуску приложения

## Быстрый перезапуск (для обычных изменений кода)

### Вариант 1: Перезагрузка на устройстве (самый быстрый)
1. На устройстве/эмуляторе встряхните телефон (shake gesture)
2. Или нажмите `r` в терминале, где запущен Metro bundler
3. Или нажмите `R` (заглавная) для полной перезагрузки

### Вариант 2: Через команду в новом терминале
```bash
# Если Metro bundler уже запущен, просто отправьте команду перезагрузки
adb shell input text "RR"
```

## Полный перезапуск (при проблемах или после изменений нативных модулей)

### Шаг 1: Остановите Metro bundler
- В терминале, где запущен Metro bundler, нажмите `Ctrl + C`

### Шаг 2: Очистите кеш и перезапустите
```bash
# Очистка кеша Metro bundler
npx expo start --clear

# Или для полной очистки
npx expo start -c
```

### Шаг 3: Перезапустите приложение на устройстве
- Если приложение уже открыто, оно автоматически перезагрузится
- Если нет, откройте приложение вручную или запустите:
```bash
npx expo run:android
```

## Полная пересборка (при серьезных проблемах)

### Для Android:
```bash
# 1. Остановите Metro bundler (Ctrl + C)

# 2. Очистите кеш и пересоберите нативный проект
npx expo prebuild --clean --platform android

# 3. Запустите приложение
npx expo run:android
```

### Для iOS (если используете):
```bash
# 1. Остановите Metro bundler (Ctrl + C)

# 2. Очистите кеш и пересоберите нативный проект
npx expo prebuild --clean --platform ios

# 3. Запустите приложение
npx expo run:ios
```

## Полезные команды Metro bundler

Во время работы Metro bundler доступны следующие команды:
- `r` - перезагрузить приложение
- `R` - перезагрузить приложение с очисткой кеша
- `m` - открыть меню разработчика
- `d` - открыть меню разработчика (альтернатива)
- `Ctrl + C` - остановить Metro bundler

## Когда использовать каждый метод

### Быстрый перезапуск (`r` или shake gesture):
- ✅ Изменения в React компонентах
- ✅ Изменения в стилях
- ✅ Изменения в логике приложения
- ✅ Изменения в TypeScript/JavaScript коде

### Перезапуск с очисткой кеша (`npx expo start --clear`):
- ⚠️ Изменения в конфигурации (app.config.js, package.json)
- ⚠️ Проблемы с загрузкой модулей
- ⚠️ Странное поведение после изменений
- ⚠️ Ошибки "module not found"

### Полная пересборка (`npx expo prebuild --clean`):
- 🔴 Изменения в нативных модулях
- 🔴 Изменения в app.config.js, влияющие на нативную часть
- 🔴 Проблемы с нативными зависимостями
- 🔴 Ошибки компиляции нативного кода
- 🔴 После установки новых нативных пакетов

## Частые проблемы и решения

### Приложение не обновляется после изменений:
1. Попробуйте `R` (заглавная) в Metro bundler
2. Или перезапустите с очисткой: `npx expo start --clear`

### Ошибки "Cannot find module":
1. Остановите Metro bundler
2. Запустите: `npx expo start --clear`
3. Если не помогло: `npx expo prebuild --clean --platform android`

### Белый экран или приложение не запускается:
1. Полная пересборка: `npx expo prebuild --clean --platform android`
2. Затем: `npx expo run:android`

### Изменения в .env.local не применяются:
1. Остановите Metro bundler
2. Перезапустите: `npx expo start --clear`
3. Если не помогло, может потребоваться пересборка нативного проекта

## Рекомендуемый рабочий процесс

1. **Для обычной разработки:**
   - Запустите Metro bundler: `npx expo start`
   - Внесите изменения в код
   - Нажмите `r` в терминале или встряхните устройство
   - Изменения применятся автоматически

2. **После изменений в конфигурации:**
   - Остановите Metro bundler
   - Запустите: `npx expo start --clear`
   - Приложение перезагрузится автоматически

3. **После установки новых пакетов:**
   - Если пакет нативный - нужна пересборка
   - Если пакет только JavaScript - достаточно перезапуска с очисткой кеша
