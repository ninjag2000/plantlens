# Как работают уведомления в PlantLens

## Обзор

В приложении используются **локальные напоминания** о уходе за растениями: полив, подкормка, опрыскивание, пересадка. Реализация — **expo-notifications**, без push-сервера: уведомления планируются на устройстве и срабатывают по таймеру.

---

## 1. Разрешения

- При первом планировании напоминания вызывается **запрос разрешения** на уведомления (`requestNotificationPermission` в `services/notificationService.ts`).
- Если пользователь отказывает, уведомления не планируются (в лог пишется сообщение, алерты не показываются).

---

## 2. Типы напоминаний

Для каждого растения хранятся до четырёх типов ухода (в `plant.reminders`):

| Ключ         | Действие   | Пример интервала по умолчанию |
|-------------|------------|------------------------------|
| `watering`  | Полив      | 7 дней                       |
| `fertilizing` | Подкормка | 14 дней                      |
| `misting`   | Опрыскивание | 2 дня                      |
| `repotting` | Пересадка  | 365 дней                     |

Частота задаётся в днях (`frequency`) и дате последней настройки (`lastSet`). Данные лежат в AsyncStorage вместе с растением.

---

## 3. Когда планируется уведомление

Уведомления **планируются** в следующих случаях:

1. **Растение добавлено в сад** — при явном добавлении в сад (PlantDetailScreen: кнопка «В мой сад»; MyPlantsScreen: кнопка «В сад» у растения в истории) вызывается **scheduleAllCareNotificationsForPlant**: пересчитываются дни до следующего ухода и планируются все четыре напоминания. Добавление растения только в коллекцию (без отдельного действия «в сад») на уведомления не влияет.
2. **Изменение частоты в настройках ухода** — при сохранении настроек в модалке «Настройки ухода» (PlantDetailScreen, `saveCareSettings`) вызывается **scheduleAllCareNotificationsForPlant**: напоминания пересчитываются с новыми частотами.
3. **Выполнение задачи** — при нажатии «Выполнено» по задаче в Care Agenda (HomeScreen, MyPlantsScreen) вызывается `scheduleCareNotification` для этого типа ухода: следующее напоминание через `freq` дней.

Перед планированием для пары «растение + тип ухода» **старое** напоминание отменяется (`cancelNotification(plantId, careType)`), затем ставится новое.

---

## 4. Логика планирования (notificationService)

- **Идентификатор** напоминания: `{plantId}-{careType}` (например `abc123-watering`). Хранится в памяти в `Map<string, string>` (id → идентификатор expo-notifications).
- **Время срабатывания:** `daysUntilDue` (дней до «следующего раза») переводится в секунды: `triggerSeconds = daysUntilDue * 24 * 60 * 60`. Если `daysUntilDue <= 0`, ставится 5 секунд (проверка «прямо сейчас»).
- **Триггер:** `TIME_INTERVAL` с `repeats: true` — одно и то же напоминание будет повторяться с выбранным интервалом.
- **Содержимое:** заголовок `PlantLens: {название растения}`, текст `Пора {действие} {название}` (сейчас фраза «Пора» захардкожена по-русски). В `data` передаются `plantId` и `careType` для возможной навигации при тапе по уведомлению.

---

## 5. Отмена

- **cancelNotification(plantId, careType)** — отменяет одно запланированное напоминание по сохранённому в памяти идентификатору. Вызывается перед повторным планированием того же типа ухода для того же растения.
- **cancelAllNotificationsForPlant(plantId)** — отменяет все четыре типа напоминаний (watering, fertilizing, misting, repotting) для растения. Вызывается при **удалении растения из сада** (PlantDetailScreen: «Удалить из сада»; MyPlantsScreen: подтверждение удаления в модалке).

---

## 6. Поведение при показе

В `setNotificationHandler` задано:

- `shouldShowAlert: true` — показывать алерт.
- `shouldPlaySound: true` — звук.
- `shouldSetBadge: false` — бейдж не меняется.

Обработка тапа по уведомлению (переход на растение/экран задачи) в коде не реализована — можно добавить через `addNotificationResponseReceivedListener` и `data.plantId` / `data.careType`.

---

## 7. Ограничения

- Идентификаторы запланированных уведомлений хранятся только в памяти (`scheduledNotifications`). После перезапуска приложения соответствие «растение + тип → идентификатор» теряется, отменить старые напоминания по коду нельзя (но они продолжат срабатывать по таймеру на устройстве).
- Текст уведомления («Пора …») не переведён на все языки приложения.
- При изменении частоты в настройках ухода напоминания пересчитываются при сохранении (saveCareSettings → scheduleAllCareNotificationsForPlant).

---

## Файлы

- **Сервис:** `services/notificationService.ts` — разрешения, планирование, отмена.
- **Использование:** `pages/HomeScreen.tsx`, `pages/MyPlantsScreen.tsx` — вызовы при завершении задачи; `pages/PlantDetailScreen.tsx` — перепланирование при сохранении настроек ухода, планирование при нажатии «В мой сад», отмена при удалении из сада; `pages/MyPlantsScreen.tsx` — планирование при нажатии «В сад» в истории, отмена при удалении из сада. Добавление в коллекцию не вызывает планирование/отмену уведомлений.
- **Данные:** `types.ts` — тип `Plant.reminders`; расчёт «когда следующий раз» и здоровья — `services/careCalculator.ts` и те же экраны (agenda, список растений).
